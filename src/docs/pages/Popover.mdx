## Usage

Use popovers for contextual actions, mini-menus, or secondary info. Keep content concise.
Avoid for critical workflows—use Modals instead. Always ensure a clear trigger.

<ComponentPreview code={`<Popover content={<div className="p-4 w-64 text-sm">Account Settings</div>}>
  <Button variant="outline">Profile</Button>
</Popover>`}>
  <div className="flex items-center">
    <Popover
      placement="bottom"
      zIndex={1}
      content={
        <div className="p-4 w-64 text-sm">
          <h4 className="font-semibold mb-2">Account Settings</h4>
          <p className="text-gray-500 mb-4 dark:text-gray-400">View and update your profile.</p>
          <Button size="sm" className="w-full">Manage Account</Button>
        </div>
      }
    >
      <Button variant="outline">Profile</Button>
    </Popover>
  </div>
</ComponentPreview>

## Anatomy

The popover is composed of two parts that live in separate parts of the DOM: a **trigger** (your `children`) and a **floating panel** (your `content`, portalled to `<body>`). An optional arrow bridges the visual gap between them.

---

## Placements & Arrows

You can control alignment and whether an arrow points directly back to the trigger.

<ComponentPreview code={`<Popover showArrow placement="right" content={<div className="p-3 text-sm">Additional detailed info goes here.</div>}>
  <Button variant="secondary">Arrow Attached</Button>
</Popover>`}>
  <div className="flex gap-4 p-8">
    <Popover
      showArrow
      placement="right"
      zIndex={1}
      content={<div className="p-3 text-sm max-w-xs">Additional detailed info goes here.</div>}
    >
      <Button variant="secondary">Arrow Attached</Button>
    </Popover>
  </div>
</ComponentPreview>

## Controlled state

Use `isOpen` + `onOpenChange` when you need external control — for instance, to close the
popover programmatically after a form submission.

<ComponentPreview code={`const [open, setOpen] = useState(false);

<div className="flex items-center gap-3">
  <Popover
    isOpen={open}
    onOpenChange={setOpen}
    content={
      <div className="p-4 w-64 text-sm">
        <p className="mb-3">This popover is fully controlled.</p>
        <Button size="sm" onClick={() => setOpen(false)}>Close</Button>
      </div>
    }
  >
    <Button variant="outline">Controlled trigger</Button>
  </Popover>
  <Button variant="ghost" size="sm" onClick={() => setOpen(true)}>
    Open from outside
  </Button>
</div>`}>
  {() => {
    const [open, setOpen] = useState(false);
    return (
      <div className="flex items-center gap-3">
        <Popover
          isOpen={open}
          onOpenChange={setOpen}
          zIndex={1}
          content={
            <div className="p-4 w-64 text-sm">
              <p className="mb-3">This popover is fully controlled.</p>
              <Button size="sm" onClick={() => setOpen(false)}>Close</Button>
            </div>
          }
        >
          <Button variant="outline">Controlled trigger</Button>
        </Popover>
        <Button variant="ghost" size="sm" onClick={() => setOpen(true)}>
          Open from outside
        </Button>
      </div>
    );
  }}
</ComponentPreview>

## Disabled state

Set `disabled` to prevent the popover from opening. The trigger is also visually muted
and non-interactive.

<ComponentPreview code={`<Popover
  disabled
  content={<div className="p-4 text-sm">You won't see this.</div>}
>
  <Button variant="outline" disabled>Disabled trigger</Button>
</Popover>`}>
  <Popover
    disabled
    zIndex={1}
    content={<div className="p-4 text-sm">You won't see this.</div>}
  >
    <Button variant="outline" disabled>Disabled trigger</Button>
  </Popover>
</ComponentPreview>

## Nested popovers

Popovers can be nested — each manages its own open state independently. Click outside the
outermost panel to close all of them. Use `zIndex` to stack panels correctly.

<ComponentPreview code={`<Popover
  placement="bottom-start"
  zIndex={10}
  content={
    <div className="p-3 w-56 text-sm">
      <p className="font-semibold mb-2">Parent popover</p>
      <Popover
        placement="right-start"
        zIndex={20}
        content={
          <div className="p-3 w-48 text-sm">
            <p className="font-semibold">Child popover</p>
            <p className="text-gray-500 mt-1">Nested content here.</p>
          </div>
        }
      >
        <Button size="sm" variant="ghost" className="w-full justify-start">
          Hover for child →
        </Button>
      </Popover>
    </div>
  }
>
  <Button variant="outline">Open parent</Button>
</Popover>`}>
  <Popover
    placement="bottom-start"
    zIndex={10}
    content={
      <div className="p-3 w-56 text-sm">
        <p className="font-semibold mb-2">Parent popover</p>
        <Popover
          placement="right-start"
          zIndex={20}
          content={
            <div className="p-3 w-48 text-sm">
              <p className="font-semibold">Child popover</p>
              <p className="text-gray-500 mt-1">Nested content here.</p>
            </div>
          }
        >
          <Button size="sm" variant="ghost" className="w-full justify-start">
            Open child →
          </Button>
        </Popover>
      </div>
    }
  >
    <Button variant="outline">Open parent</Button>
  </Popover>
</ComponentPreview>
